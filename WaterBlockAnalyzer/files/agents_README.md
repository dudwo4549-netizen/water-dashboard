```

├── CLAUDE.md                          ← 전체 프로젝트 지침
└── .claude/
    └── agents/
        ├── README.md                  ← 이 파일
        ├── seoyong.md                 ← 서용이 부장
        ├── cheongryong.md             ← 청룡 대리
        ├── dongryong.md               ← 동룡 기술사
        ├── okryong.md                 ← 옥룡 분석가
        └── images/
            ├── seoyong/               ← 서용이 표정 이미지 (16개)
            ├── cheongryong/           ← 청룡 표정 이미지 (2개)
            ├── dongryong/             ← 동룡 표정 이미지 (1개)
            └── okryong/               ← 옥룡 표정 이미지 (1개)
```

### 2. Claude Code 실행

```bash
cd D:/Projects/WaterBlockAnalyzer
claude
```

### 3. 첫 호출

```
> 서용이 사단, 출동! 오늘부터 본격적으로 단계시험 플러그인 만들자.
```

Claude Code가 자동으로 서용이를 호출하고, 필요에 따라 다른 팀원을 부릅니다.

## 💡 사용 패턴

### 패턴 1: 명시적 호출

특정 전문가에게 직접 작업 요청.

```bash
> @okryong QGIS에서 단계시험 분석 코드 작성해줘
> @cheongryong 이 다이얼로그 UI 좀 다듬어줘
> @dongryong 누수율 50/100 기준이 환경부 매뉴얼이랑 맞는지 검토해줘
```

### 패턴 2: 서용이를 통한 위임

서용이가 작업을 받아서 적절한 팀원에게 배분.

```bash
> @seoyong 수압 분석 플러그인 만들고 싶어. 어떻게 진행할까?
```

→ 서용이가 기획하고, 옥룡(분석 로직), 청룡(UI), 동룡(기준 검증)을 순차 호출

### 패턴 3: 자동 위임 (Auto-delegation)

작업 성격에 맞춰 Claude Code가 자동으로 적절한 에이전트 선택.

```bash
> 누수이력 엑셀 파일을 QGIS 포인트 레이어로 변환하는 코드 짜줘
```

→ Claude Code가 자동으로 `@okryong` 호출 (GIS 작업)

### 패턴 4: 다중 협업

여러 에이전트가 한 작업에 협업.

```bash
> 간접평가 보고서 자동 생성 시스템 만들자.
  옥룡이 분석 로직, 청룡이 UI, 동룡이 평가 기준 검증해줘.
```

## ⚙️ 동작 원리

### Claude Code의 에이전트 시스템

1. **자동 발견**: `.claude/agents/*.md` 파일을 자동으로 인식
2. **컨텍스트 분리**: 각 에이전트는 독립된 컨텍스트로 작업
3. **자동 위임**: 작업 성격에 맞춰 적절한 에이전트 자동 호출 가능
4. **명시적 호출**: `@에이전트명` 형식으로 직접 호출 가능

### YAML 헤더 의미

```yaml
---
name: seoyong              # 에이전트 식별자 (@호출명)
description: ...           # 언제 호출되는지 설명 (자동 위임 판단)
tools: Read, Write, ...    # 사용 가능한 도구
---
```

### 새 에이전트 추가

`.claude/agents/` 폴더에 새 .md 파일 추가:

```markdown
---
name: new_agent
description: 새 전문가의 역할
tools: Read, Write
---

# 에이전트 내용
```

### 기존 에이전트 수정

해당 .md 파일을 편집하면 즉시 반영됩니다. Git으로 버전 관리 권장.

### 이미지 추가

`.claude/agents/images/[에이전트명]/` 폴더에 PNG 파일 추가 후, 해당 에이전트 .md에서 경로 등록.

## 💬 자주 묻는 질문 (FAQ)

**Q1. 안티그라비티 버전과 다른 점이 있나요?**
- 기본 페르소나/말투/역할은 100% 동일
- 추가로 Claude Code 환경에 맞는 도구 사용 권한, CLAUDE.md 규칙 준수 명시
- 이미지 경로가 `images/`에서 `.claude/agents/images/[이름]/`으로 변경

**Q2. 이미지가 표시 안 돼요**
- Claude Code는 터미널 환경이라 이미지가 인라인 렌더링되지 않을 수 있습니다
- 이미지는 주로 문서/보고서 출력 시 활용됩니다
- VS Code 등 IDE의 Markdown 미리보기에서는 정상 표시됩니다

**Q3. 4명 다 동시에 호출하면?**
- Claude Code는 한 번에 한 에이전트가 응답하지만, 작업 분담은 자동으로 조율됩니다
- 서용이를 통하면 자연스러운 위임이 가능합니다

**Q4. 안티그라비티에서도 같이 쓸 수 있나요?**
- 네. 안티그라비티의 원본 `instructions.md`는 그대로 두시고, Claude Code용 .md만 추가하면 됩니다
- 두 환경에서 동일한 캐릭터로 작업 가능
